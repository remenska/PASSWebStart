Property Pattern Mappings for mu-calculus
These Property Pattern mappings for mu-calculus are an extension of the original mappings available at the CADP webpage. The extensions are marked in green. There are 4 scope and 2 behavior variations leading to more than 100 unique patterns to be chosen from.
Below is a brief explanation on the extensions and their usefulness in particular situations.

Note: For certain properties (liveness), alternatives are given with and without a fairness assumption: it is the assumption that the system scheduling is fair, i.e., a process cannot continously ask to make progress without being allowed to do so, eventually.

We advise using PASS to automatically obtain the correct formula for a property, from this pattern system.
Before R variant:
If R does not occur until the end of the event sequence, the behavior is required to hold throughout the entire event sequence. ("Until R")
After Q variant:
If there are multiple occurrences of Q before the end of the event sequence, only the last occurrence starts the restricted interval; each consecutive occurrence of Q resets the beginning of the restricted interval.
Between Q and R variant:
If there are multiple occurrences of Q before the end of the event sequence, only the last occurrence starts the restricted interval; each consecutive occurrence of Q resets the beginning of the restricted interval. If R does not occur until the end of the event sequence, the behavior is not required to hold anywhere in the event sequence.
After Q until R variant:
If there are multiple occurrences of Q before the end of the event sequence, only the last occurrence starts the restricted interval; each consecutive occurrence of Q resets the beginning of the restricted interval. If R does not occur until the end of the event sequence, the behavior is required to hold throughout the entire event sequence.
Precedence variant:
S is not allowed to occur until P has occurred. In other words, P must precede S. In addition, S is required to occur, whether or not P eventually occurs.
Response variant:
If P occurs, then S is required to occur subsequently. In other words, S must respond to P. In addition, P is required to occur throughout the execution sequence.
